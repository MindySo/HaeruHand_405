🚀 카카오 OAuth 로그인 테스트 가이드

실제 유저 플로우를 따라 카카오 로그인을 테스트합니다

📋 사전 준비사항

1

환경변수 설정

다음 환경변수가 설정되어 있어야 합니다:

CLIENT_ID=your_kakao_app_key JWT_SECRET=your_jwt_secret_key SERVER_PORT=8080 MYSQL_HOST=localhost MYSQL_PORT=3306 MYSQL_DATABASE=haeruhand MYSQL_USER=root MYSQL_PASSWORD=your_password REDIS_HOST=localhost REDIS_PORT=6379
2

카카오 앱 설정

카카오 개발자 콘솔에서 다음 설정을 확인하세요:

  • Redirect URI: http://localhost:8080/callback.html
  • 동의항목: 프로필 정보(닉네임/프로필 사진) 필수 동의
3

index.html 수정

index.html 파일에서 KAKAO_CLIENT_ID를 실제 값으로 변경:

const KAKAO_CLIENT_ID = 'YOUR_ACTUAL_KAKAO_APP_KEY';

🏃 테스트 실행 방법

1

백엔드 서버 실행

cd backend/spring-business/haeruhand ./gradlew bootRun

서버가 http://localhost:8080에서 실행됩니다.

2

로그인 페이지 접속

브라우저에서 다음 URL로 접속:

http://localhost:8080/index.html
3

카카오 로그인 진행

  1. "카카오 로그인" 버튼 클릭
  2. 카카오 계정으로 로그인
  3. 권한 동의 화면에서 "동의하고 계속하기" 클릭
  4. 자동으로 callback.html로 리다이렉트
  5. 토큰 발급 완료 후 index.html로 이동

🧪 API 테스트

1. 토큰 발급 (카카오 인증코드 필요)

POST /v1/user/issue/kakao
{ "code": "KAKAO_AUTHORIZATION_CODE" }

2. 사용자 정보 조회

GET /v1/user/userinfo
Headers: Authorization: Bearer YOUR_ACCESS_TOKEN

3. 토큰 재발급

POST /v1/user/reissue
Headers: X-Refresh-Token: YOUR_REFRESH_TOKEN

📊 테스트 시나리오

시나리오 기대 결과 확인 방법
신규 사용자 로그인 사용자 생성 및 토큰 발급 DB에 새 사용자 레코드 확인
기존 사용자 재로그인 기존 사용자 조회 및 토큰 발급 사용자 ID 동일한지 확인
액세스 토큰 만료 401 Unauthorized 응답 1시간 후 API 호출
리프레시 토큰으로 갱신 새 액세스 토큰 발급 /reissue 엔드포인트 호출
토큰 초기화 로그아웃 상태 localStorage 확인

🐛 문제 해결

⚠️ CORS 에러 발생 시

WebConfig.java에서 CORS 설정을 확인하세요. localhost:8080이 허용되어 있어야 합니다.

⚠️ 카카오 로그인 실패 시
  • CLIENT_ID가 올바른지 확인
  • Redirect URI가 정확히 일치하는지 확인
  • 카카오 앱이 활성화 상태인지 확인
⚠️ 토큰 발급 실패 시
  • JWT_SECRET 환경변수 확인
  • Redis 서버 실행 상태 확인
  • 백엔드 로그에서 에러 메시지 확인

✅ 테스트 완료 체크리스트

  • ☐ 카카오 로그인 페이지 정상 표시
  • ☐ 카카오 계정으로 로그인 성공
  • ☐ 액세스 토큰 발급 확인
  • ☐ 리프레시 토큰 발급 확인
  • ☐ 사용자 정보 조회 성공
  • ☐ 토큰 갱신 성공
  • ☐ 로그아웃 (토큰 초기화) 성공
🎉 모든 테스트가 완료되면

실제 프론트엔드 애플리케이션과 통합할 준비가 완료됩니다!

redirectUri를 실제 프론트엔드 URL (예: http://localhost:3000/callback)로 변경하세요.
